StyleSpy reads the CSS of every element on a page — including the hidden ones and the hover/tooltip states that normally vanish — and gives you copy-ready selectors for QA automation. Here's how everything works.

Quick start

  1. Open any web page and click the StyleSpy icon to open the side panel.
  2. Click + Capture Page. StyleSpy scans the page and lists every element with its CSS.
  3. Click any row to expand it, or use Full table ↗ to see everything in a wide, exportable grid.

The toolbar (top of the panel)

❄ Freeze ⌘/Ctrl+Shift+F

Locks the page's current :hover, :focus and :active states and pauses animations — so a tooltip, dropdown or menu stops disappearing and you can inspect it. Press the shortcut while hovering the thing you want (the keyboard doesn't move your mouse, so the state is caught). Click Unfreeze to release.

⌖ Pick ⌘/Ctrl+Shift+E

Click-to-inspect. A highlight box follows your cursor; click any element and StyleSpy jumps to the Inspect tab with its full details. Press Esc to stop picking.

⏱ 3s capture

Starts a 3-second countdown, then freezes automatically — giving you time to open a menu first, then freeze it so you can read its styles.

The tabs

Capture

The main view. + Capture Page scans the page; the list shows a colour swatch, tag, text and key properties per element. Click a row to expand full properties plus copy buttons for the CSS selector and XPath. Filter with the chips:

  • all — everything captured (the default scope grabs every element, so expect layout divs too).
  • visible — only on-screen, rendered elements (use this to cut noise).
  • hidden — only elements hidden by CSS.
  • dynamic — elements captured from a frozen/picked state.

The search box matches text, selector, tag, classes and colour (e.g. type #2f7d72).

Inspect

A deep view of a single element (from Pick or Freeze): typography, the box model, a text → background colour pair, the WCAG contrast ratio with an AA pass/fail flag, and both locators. Copy CSS rule gives you a ready-to-paste rule block.

Force

Force :hover / :focus / :active on a CSS selector without a real mouse. StyleSpy reads the page's own stylesheets and re-applies the matching state rules, so you can study a styled state hands-free. Pick an element first to pre-fill its selector.

Ghost

Finds elements hidden via display:none, visibility:hidden, opacity:0 or zero size — closed modals, un-triggered dropdowns, tracking pixels, screen-reader-only text. Click the 👁 eye to force-reveal a node (it gets a dashed outline); click again to re-hide.

Mutations

A live feed of DOM changes (added / removed / attribute / text). Turn on Break on next mutation to freeze the page the instant a new node appears — perfect for catching a dropdown's markup as it opens.

The full table

Full table ↗ opens your capture in a new browser tab as a wide spreadsheet: sortable columns, show/hide columns, colour chips, visibility flags, and the same export options. It's row-virtualized, so thousands of elements stay smooth.

Locator health for QA

After a capture, the Capture tab shows a Locator health bar that grades the locators of the page's visible interactive elements (buttons, links, inputs, and anything with an interactive ARIA role) — the things a test actually clicks or types into. Hidden nodes and layout divs are excluded, so the score reflects real test-readiness. You can see at a glance how much of a screen is safely targetable by Selenium / Playwright / Cypress, and which parts need better hooks. Click any coloured segment to filter the list to just those elements. The full table has a matching Locator column (graded per element).

strong A data-testid (or data-test/data-qa) or a unique id. Stable across redesigns — the gold standard.
class A meaningful, human-authored class, e.g. a.hero-cta. Usable, but can change with styling.
fragile Deep positional XPath like //*[@id='root']/div[1]/div[1]/a[1]/img[1], framework/utility classes only like div.MuiContainer-root, hashed css-1a2b3c names, or :nth-of-type() chains. These break easily — add a data-testid.

The “% automatable” figure is the share of visible interactive elements that have a strong locator. Use it as a quick test-readiness score for a screen.

Export, clear & theme

  • Export CSV / Export JSON — download the current (filtered) capture.
  • Copy — copy the capture as JSON to the clipboard.
  • Clear — wipe the current capture from the panel and the table.
  • ☾ / ☀ (top right) — switch between light and dark; your choice is remembered.
  • Your last capture is restored automatically when you reopen the panel.

Shortcuts can be changed at chrome://extensions/shortcuts.

Tips

  • Capturing "every element" is thorough but noisy — hit the visible filter to focus on what's on screen.
  • Force/Freeze work on CSS-driven states. Tooltips built purely in JavaScript may need Freeze (which snapshots the live state) rather than Force.
  • Selectors prefer data-testid / id — ideal for Selenium, Playwright and Cypress.
